# 关于地形生成 guideline 的理解

这份 guideline 的核心目标，是把地图生成重写成一个可控、有限边界、由种子驱动的程序化生成流程。当前阶段先不追求完整世界系统，而是先把“地表 + 材质层 + 简单洞穴”这条主链路做出来。

## 1. 数学工具层

首先需要先实现一组噪音函数，作为后续地图生成的基础工具：

- 一维 Perlin 噪音函数。
- 二维 Perlin 噪音函数。
- 噪音输入需要包含随机数种子、Perlin 基本参数，以及输出尺寸。
- 二维 Perlin 噪音还需要支持旋转，用来避免洞穴或地形纹理只沿水平/垂直方向出现规律感。

我理解这里的重点不是只写一个固定噪音，而是写成可复用的数学模块。后续地平线、泥土分布、洞穴形状都可以用不同参数调用同一套工具。

## 2. 世界边界

地图不是无限加载的区块世界，而是一个有明确边界的有限地图。

当前地图范围为：

- x: -2000 到 2000。
- y: -700 到 300。
- y = 0 作为基础平面。

这意味着地图生成可以按完整二维区域一次性生成，也可以按内部数组/网格生成后缓存下来，不需要设计无限区块加载、卸载、流式生成这些系统。

## 3. 地平线生成

第一步是生成地表高度线，也就是每个 x 位置对应的地表 y 值。

地表高度线的生成逻辑可以分成两层：

- 用多个正弦波叠加，形成主要的地形起伏。
- 再叠加几个较小的一维 Perlin 噪音，让地表不那么机械，增加局部细节。

我的理解是：

- 正弦波负责“大轮廓”，比如缓坡、丘陵、长周期起伏。
- 一维 Perlin 噪音负责“小扰动”，让地表边缘更自然。
- 最终得到的结果应该是一个 `heightAtX(x)` 或等价的高度数组。

生成地平线之后，可以先用最基础的规则填充世界：

- 地表以上为空气。
- 地表以下先填充为石头。

## 4. 地表附近材质转换

在基础石头地形生成完成后，需要用二维噪音处理地表附近的材质。

规则是：

- 靠近地表的一部分石头，会被二维噪音不规则地转换为泥土。批注：10格左右的泥土
- 这种转换不能是一条固定厚度的水平泥土层，而应该有自然的厚薄变化。
- 如果一个泥土方块上方没有方块，则把这个泥土方块变为草方块。

我理解这里想要的是类似这样的层次：

- 深层主要是石头。
- 接近地表处混入不规则泥土。
- 最外层裸露泥土自动变草。

二维噪音用于让泥土层边界不平整，避免出现“固定 3 格泥土厚度”这种太规则的效果。

## 5. 简单洞穴生成

洞穴生成基于两个二维 Perlin 噪音场的叠加/差值判断。

需要生成两个噪音值：

- A 噪音：间隙较大、幅度较大、多个叠加，用来控制整体洞穴是否产生。
- B 噪音：间隙较小、幅度较小、多个叠加，用来控制洞穴边界的不规则细节。

两个噪音都可以施加一定旋转。

判断逻辑是：

- 对每个地图位置计算 `A - B`。
- 如果差值大于某个阈值/比例，则该位置变成空气。

我的理解是：

- A 决定“大洞在哪里”，也就是洞穴主体分布。
- B 从 A 中扣掉一部分细节，让洞穴边缘更破碎、更自然。
- `A - B > threshold` 的区域被挖空，形成洞穴。

洞穴生成应该发生在基础地形填充之后。也就是说，先生成石头/泥土/草，再根据洞穴规则把部分实体方块替换为空气。

## 6. 当前阶段的边界

“先做到这里为止”表示当前重写的第一阶段只需要完成：

- 有限地图范围。
- 一维/二维 Perlin 噪音工具。
- 基于正弦波和一维噪音的地表高度线。
- 地表以下基础填充。
- 靠近地表的泥土和草方块生成。
- 基于双二维噪音差值的简单洞穴生成。

暂时不需要处理：

- 无限区块加载。
- 生物群系。
- 矿物。
- 建筑/遗迹。
- 水、岩浆等流体。
- 树木、植物和装饰物。
- 敌人、宝箱或玩法内容。

## 7. 我对整体生成顺序的理解

推荐的第一版生成管线是：

1. 根据种子初始化随机和噪音系统。
2. 在固定地图边界内建立二维方块数组。
3. 对每个 x 计算地表高度。
4. 根据地表高度填充空气和石头。
5. 对地表附近区域使用二维噪音，把部分石头转换为泥土。
6. 扫描泥土方块，如果上方为空气，则转换为草方块。
7. 对地下区域计算 A、B 两组二维噪音。
8. 当 `A - B` 超过阈值时，将对应实体方块挖空为空气。

这个顺序的好处是每一步都很清楚：先确定形状，再确定材质，再挖洞。后续如果要加矿物、植被、结构，也可以继续插入到这个管线里。
